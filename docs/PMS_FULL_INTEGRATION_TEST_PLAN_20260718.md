# Hotel PMS 전체 통합 테스트 계획서

- 문서 버전: 1.0
- 기준일: 2026-07-18
- 대상: Hotel PMS Dashboard, Super Admin, 공통 API/저장소/다국어/반응형 동작
- 기준 문서: `docs/PROCESS_TEST_STANDARD.md`
- 테스트 원칙: 실제 페이지에서 사용자가 수행하는 순서로 입력, 저장, 재진입, 교차 화면 반영까지 확인한다.

## 1. 목적

이 문서는 현재 구현된 모든 Hotel PMS 및 Super Admin 페이지의 기능을 테스트 케이스로 고정하고, 이후 수정과 회귀 테스트를 동일한 기준으로 수행하기 위한 실행 계획서다.

다음 조건을 모두 만족해야 해당 기능을 Pass로 판정한다.

1. 실제 화면에서 버튼, 입력, 선택, 팝업을 사용자 순서대로 조작할 수 있다.
2. 등록, 수정, 삭제, 상태 변경 결과가 현재 화면에 즉시 반영된다.
3. 팝업 재오픈, 메뉴 이동 후 복귀, 새로고침 후에도 결과가 유지된다.
4. 관련된 다른 화면의 목록, 카운트, 상태, 금액, 로그에도 같은 결과가 반영된다.
5. 한국어와 영어에서 라벨, placeholder, select option, 팝업, 알림, 브라우저 제목이 섞이지 않는다.
6. 데스크톱, 태블릿, 모바일에서 클릭, 터치, hover, dropdown, modal이 안정적으로 동작한다.
7. 예상하지 않은 console error, page error, 4xx/5xx 응답이 없다.

### 2.3 테스트 제외 기준

- `old/`, `sample/`, `outputs/backups/`, `manual/`은 제품 실행 화면이 아니므로 기능 통합 테스트 대상에서 제외한다.
- 직접 URL로만 접근 가능한 호환 화면도 `dashboard/` 또는 `admin/` 아래의 현재 HTML이면 테스트한다.
- 외부 결제, 메일, 실제 프린터처럼 실연동이 없는 기능은 요청 payload, 화면 상태, fallback, 오류 처리까지 검증하고 실서비스 연동 결과는 Blocked로 분리한다.

## 5. 판정 및 증거 기준

| 판정 | 기준 |
| --- | --- |
| Pass | 절차, 화면, 저장, 재진입, 교차 화면, 로그가 모두 기대 결과와 일치 |
| Fail | 한 단계라도 기대 결과와 다르거나 console/network 오류 발생 |
| Blocked | 외부 연동, 권한, 필수 데이터 부재로 실행 불가. 원인과 해제 조건 필수 기록 |
| Not Run | 아직 실행하지 않음 |

각 케이스는 다음 증거를 남긴다.

- 실행 전 상태 캡처
- 입력값 또는 선택값
- 실행 직후 화면 캡처
- 새로고침/재진입 후 화면 캡처
- 관련 화면 또는 감사 로그 캡처
- console error 및 실패한 request 목록
- 실행 환경, 언어, viewport, 사용자 역할

| --- | --- | --- |
| Blocker | 로그인 불가, 데이터 손실/오염, 다른 Tenant 노출, 결제/정산 중복 | 즉시 중단 |
| Critical | 핵심 등록/체크인/체크아웃/정산/마감 프로세스 실패 | 배포 불가 |
| Major | 기능은 가능하나 상태/카운트/로그/언어/모바일이 잘못됨 | P1 포함 수정 후 재검증 |
| Minor | 의미에 영향 없는 시각적 차이 | 영향도 합의 후 추적 가능 |

### 13.2 완료 게이트

최종 완료는 다음 조건을 모두 만족할 때만 선언한다.

1. Blocker, Critical, Major 미해결 결함이 0건이다.
2. 모든 P0 케이스가 Pass다.
3. 모든 P1 케이스가 Pass이거나, 승인된 예외와 후속 일정이 기록되어 있다.
4. 수정된 케이스뿐 아니라 영향받는 모듈의 공통·E2E·다국어·반응형 회귀가 Pass다.
5. 테스트하지 않은 기능을 성공으로 간주하지 않는다. 실행하지 않은 케이스는 반드시 `Not Run`이다.
6. 실행 commit, 배포 URL, fixture, 역할, 언어, viewport, 증거 링크가 리포트에 남아 있다.

## 17. 기준 사용 원칙

- 기능 추가 시 해당 페이지 전용 케이스, 교차 모듈 케이스, 데이터 케이스를 함께 추가한다.

- 테스트 실행 보고서에는 이 문서의 전체 Case ID 대비 Pass/Fail/Blocked/Not Run 수를 기재한다.
- `모두 성공`이라는 표현은 13.2 완료 게이트를 충족하고 증거가 연결된 경우에만 사용한다.
