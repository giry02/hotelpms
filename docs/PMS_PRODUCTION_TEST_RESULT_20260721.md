## 3. 이번 실행에서 보정한 테스트 코드

운영 화면 제품 결함은 남은 실행에서 새로 재현되지 않았다. 다만 과거 데이터나 브라우저 구현에 의존해 잘못 실패하거나 멈추던 자동화 5개를 수정했다.

1. 신규 예약 테스트의 고정 날짜를 제거하고 실행 당일 기준 날짜를 생성하도록 변경했다.
2. 예약 취소를 확인 창과 필수 취소 사유 입력의 2단계 절차로 실행하도록 변경했다.
3. 단체 객실 배정 테스트가 화면 재렌더링 뒤의 실제 활성 옵션을 다시 선택하도록 변경했다.
4. 하우스키핑·시설 삭제 테스트가 기본 `confirm`뿐 아니라 실제 비동기 확인창 `showConfirm`도 처리하도록 변경했다.
5. 지원 중단된 모바일 폭 대신 최소 지원 태블릿 폭 820px에서 레이아웃 안정성을 검사하도록 변경했다.

- 전체·단체 필터 간 동일 카드 상태와 색상 유지

## 5. 시각 감사 시간 초과 의미

시간 초과는 화면이 깨졌다는 판정이 아니다. 기존 통합 실행은 60페이지를 4개 뷰포트로 검사해 한 프로세스에서 최대 240번의 페이지 로드와 캡처를 수행했고, 명령 실행 제한시간을 넘겼다. 현재 지원 범위인 데스크톱·태블릿만 대상으로 페이지군을 분리해 다시 실행했으며 총 120건이 실패와 경고 없이 완료됐다.
