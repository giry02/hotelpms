### 3.2 예외 경로

같은 객실에 과거 종료된 `checkedin` 예약을 의도적으로 삽입한 뒤 신규 투숙 예약을 생성했다. 수정 전에는 부가서비스가 신규 예약 대신 과거 예약 `RSV-E2E-STALE-9908`에 연결되어 실패했다.

| 시점 | 예상 | 실제 | 판정 |
| --- | --- | --- | --- |
| 수정 전 | 신규 예약 ID 연결 | 과거 종료 예약 ID 연결 | FAIL 재현 |
| 수정 후 | 신규 예약 ID 연결 | `RSV-20260722104109-703` 연결 | PASS |

원인은 부가서비스 화면이 객실의 활성 예약을 찾을 때 체크아웃 날짜를 검사하지 않고 `checkedin` 상태만 우선한 로직이었다. 실행일이 숙박 기간 안에 있는 예약만 허용하고, 여러 건이면 가장 최근 체크인 예약을 선택하도록 수정했다.

## 6. 차단 사항

### CONC-001 동시 수정 충돌

정적 `localStorage`에는 레코드 버전, 서버 트랜잭션, ETag 또는 조건부 갱신이 없다. 따라서 두 사용자가 같은 데이터를 수정할 때 마지막 저장이 앞선 저장을 덮어쓴다. 이 항목은 백엔드 도입 전에는 해결 완료로 판정할 수 없다.

### ADM-USER-008 서버 권한 우회

현재 배포본은 권한을 검증하는 실제 서버 API가 없다. 화면 접근 제어는 확인했지만, 권한 없는 사용자의 직접 API 호출이 403으로 거절되는지는 검증할 대상 자체가 없다.

## 8. 결론

우선순위 1·2·5는 운영 서버에서 동일 데이터로 연결 실행했고, 실제 결함 1건을 재현·수정·배포·재검증했다. 남은 기능 실패는 0건이다. 다만 동시 편집 충돌과 서버 API 권한 검증은 현재 정적 구조의 한계로 차단되어 있으므로, 전체 시스템을 무결점 또는 서버 운영 준비 완료로 판정하지 않는다.
